# プロトタイプ検証: 紙プレイテスト手順書

> 入力: docs/01_planning.md, docs/02_requirements.md, data/cards_draft.csv,
> prototype/print_cards.html

## 1. 準備するもの

- prototype/print_cards.html を印刷したカード24枚（攻撃16／防御4／強化4）
- 対戦相手キャラクターシート2種（性格タイプと弱点属性を裏に記載したメモで代用可）
  - 例: キャラX＝短気（弱点属性: 煽り）、キャラY＝繊細（弱点属性: 皮肉）
- 誇りゲージ・怒りゲージを表すトークン（コイン等で代用可、各20点分）
- ターン記録用の簡易メモ用紙

## 2. プレイ手順

1. プレイヤーは16枚の攻撃カードから10枚、防御4枚、強化4枚の中から任意枚数を選び、
   合計10〜20枚でデッキを組む（docs/02_requirements.md のデッキ枚数上限に準拠）。
2. CPU役（もう1人のプレイヤーが代行）はキャラX（短気・弱点=煽り）を担当する。
3. 誇りゲージは双方20からスタート。
4. 1ターンにつきコスト3を上限として手札からカードを使用する。
5. 攻撃カード使用時、カードの属性がキャラXの弱点属性（煽り）と一致する場合は
   カード記載の追加ダメージを適用する。
6. 誇りゲージが0になった時点でそのゲームは終了。
7. 同じ手順をキャラY（繊細・弱点=皮肉）相手にも1回行う（計2ゲーム）。

## 3. 観察するポイント

- **弱点属性の推測は成立するか**: プレイヤーは「短気」「繊細」といった性格ラベルから
  弱点属性を直感的に予測できたか、それとも表記を見ないと分からなかったか。
- **二重リソースの読み合いは機能するか**: 「今すぐ誇りを削るか、怒りを溜めて
  強力な一撃を待つか」という選択に悩む場面が実際に発生したか。
- **決着までのターン数**: 何ターンで誇りゲージが0になったか（早すぎ・長すぎの判断材料）。
- **手詰まり／一方的な展開の有無**: コスト不足で何もできないターンが続かないか、
  逆に強力なカードの連発だけで決着してしまわないか。
- **防御・強化カードの存在感**: 攻撃カードばかり使われて防御・強化カードが
  一度も選ばれない、という偏りが起きていないか。

## 4. 勝敗・記録の判定方法

- 各ゲームについて「勝者」「決着までのターン数」「使用された攻撃カードの属性内訳」
  「防御・強化カードの使用回数」をメモに記録する。
- プレイ後、プレイヤー自身の感想（「どこが気持ちよかったか」「どこが退屈だったか」）を
  一言メモとして残す。

## 5. 次アクション

上記のプレイを最低2ゲーム（キャラX戦・キャラY戦）実施した後、その結果メモを
docs/03_prototype_findings.md に反映する。人間による物理プレイテストが
実施できない場合は、本手順書のロジックに基づく卓上シミュレーション分析で代替し、
その旨を findings.md に明記した上で設計フェーズに進む。
