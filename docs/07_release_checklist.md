# リリースチェックリスト

> 入力: docs/01〜06 の全成果物

## 前提の断り書き

本プロジェクトは1セッション内でClaude Codeが自律的に企画〜テストまでを一気通貫で
実施したものであり、**実際の人間によるプレイテストは未実施**である
（docs/03_prototype_findings.md、docs/06_balance_report.md 参照）。
数値バランスはロジック上のシミュレーションでは目標レンジ内に収まっているが、
「実際に遊んで面白いか」は依然として未検証のため、公開前に必ず人間によるプレイテストを
実施すること（docs/backlog.md の最優先項目）。

## 1. ビルド手順とデプロイ先

- ビルド: `npm run build`（tsc -b && vite build。`dist/` に静的アセットを出力）。
- デプロイ先候補: Vercel または Netlify。
  - 理由: 本プロジェクトは静的ビルドのReact SPA（バックエンドAPI不要）であり、
    どちらも `dist/` をそのままホスティングでき、個人開発でも無料枠で運用できるため。
  - Gitリポジトリ連携によるPushからの自動デプロイが可能な点も、ソロ開発での
    運用負荷を下げる理由になる。

## 2. バージョニング方針

- セマンティックバージョニング（MAJOR.MINOR.PATCH）を採用する。
- MVP初回リリースは `0.1.0` とする（1.0.0はマルチプレイ等の主要機能追加後を想定）。
- バランス数値のみの調整（docs/04_balance_rules.mdの変更のみ）はPATCH更新とする。
- 新カード追加・新メカニクス（パターンB/C）追加はMINOR更新とする。

## 3. リリース後のフィードバック収集方法

- アプリ内に「感想・不具合報告」への簡易リンク（外部フォーム等）を設置する
  （実装はMVPスコープ外だが、リンク設置自体は軽量なため次バージョンで対応）。
- 個人開発のため、SNS等での告知に対するリプライ・引用も一次情報として収集する。
- 収集したフィードバックは、docs/06_balance_report.md や本チェックリストと同様に
  `docs/` 配下に蓄積し、次バージョンのdocs/01_planning.mdの見直し材料とする。
