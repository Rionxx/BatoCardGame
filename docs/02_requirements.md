# 要件定義: 罵倒デッキ構築カードゲーム

> 入力: docs/01_planning.md
> このファイルは後工程で頻繁に参照するため、曖昧な表現を避け断定形で記述する。

## 1. MVPスコープ

### MVPに含める機能

- シングルプレイ・対CPU戦のみ。
- 戦闘は「1戦＝相手1人との対決」を最小単位とし、3戦勝ち抜きの1ラン構成とする。
- 罵倒メカニクスは docs/01_planning.md の**パターンA（属性相性システム）のみ**を実装する。
- 二重リソース制（誇りゲージ／怒りゲージ）を実装する。ただし怒りゲージの使途は
  「一定値到達で強力な罵倒カードが1枚解禁される」という単純なトリガー止まりとする。
- 対戦相手の直近の言動ログ（行動履歴のテキスト表示）を戦闘画面に表示し、
  プレイヤーが弱点属性を推測する材料とする。
- 勝利後にカードを1〜3枚の候補から1枚選んでデッキに加える、簡易デッキ構築を実装する。
- 日本語UIのみ。テキストとイラスト（プレースホルダー可）によるカード表現。
- ブラウザ内のみで完結し、リロードでランが失われても良い（セーブ機構なし）。

### MVPに含めない機能

- マルチプレイ（対人対戦）。
- パターンB（動揺ゲージ／ノイズカード混入）、パターンC（連続罵倒コンボ／決め台詞デッキ）。
- 課金・収益化要素。
- モバイル操作（タッチ）の最適化。
- 音声・ボイス表現。
- セーブ／クラウド同期。
- アチーブメント・実績システム。
- 丁寧なチュートリアル（操作説明は簡易テキストのみ）。
- 多言語ローカライズ。

## 2. 非機能要件

- **対応ブラウザ**: PC版 Chrome / Edge / Safari の最新版を正式サポート対象とする。
  モバイルブラウザは動作確認のみ行い、UI崩れの修正はMVPスコープ外とする。
- **オフライン対応**: 不要（Web配信前提であり、オフラインでのプレイは想定しない）。
- **想定同時プレイ人数**: 1人（シングルプレイ・CPU対戦のみのため）。
- **パフォーマンス**: 高負荷な演算・描画は発生しない想定のため、一般的なノートPCで
  UI操作にストレスのない応答速度（体感で操作から100ms以内の反応）を目標とする。
- **アクセシビリティ**: 色弱対応・スクリーンリーダー対応は将来課題とし、MVPスコープ外とする。

## 3. カードゲームとして最低限必要なデータ（仮決定）

- **カード種別**: 攻撃（罵倒）カード／防御（受け流し）カード／強化（バフ）カードの3種。
  決め台詞カード（パターンC由来）はMVPでは実装しない。
- **リソース系統**:
  - コスト（手番内で使用できるアクションポイント。ターン開始時に一定量回復）
  - 誇りゲージ（0になった側が敗北。攻撃カード命中で減少）
  - 怒りゲージ（攻撃カードを使用した側が蓄積。一定値到達で強力な罵倒カードが1枚解禁）
- **属性**: 罵倒カードは「煽り／正論／皮肉／自虐からの逆転」のいずれかの属性を持つ。
  対戦相手キャラクターは「短気／プライド過剰／繊細／鈍感」のいずれかの性格タイプを持ち、
  対応する弱点属性が1つ設定される。
- **勝利条件**: 相手の誇りゲージを0にする。
- **デッキ枚数上限**: 最小10枚、上限20枚（仮決定。設計フェーズでバランス数値とあわせて確定する）。

## 4. スコープ外項目とその理由

| 項目 | スコープ外にした理由 |
|---|---|
| マルチプレイ | 弱点推測メカニクスが対人前提での再設計を要し、MVPの検証目的（属性相性が面白いか）から外れるため |
| パターンB（動揺ゲージ） | パターンAのみで面白さが成立するかを先に検証する必要があるため |
| パターンC（連続コンボ） | 同上。段階的追加の方針を企画フェーズで既に決定済みのため |
| 課金・収益化 | 個人開発における検証目的のプロジェクトであり、収益化はリリース後の判断事項であるため |
| モバイル最適化 | 開発リソースが限られるソロ開発のため、まずPCブラウザでの体験確立を優先するため |
| 音声・ボイス | テキストベースでコアファンタジーが成立するかをまず検証するため、演出コストを抑える |
| セーブ／クラウド同期 | 1ランが30〜45分で完結する設計のため、MVPでは必須ではないため |
| アチーブメント | 継続動機の強化施策であり、コアループが未検証の段階では優先度が低いため |
| 丁寧なチュートリアル | ルールの複雑さが確定していない段階で作り込むと手戻りが大きいため |
| 多言語ローカライズ | 個人開発・日本語圏での検証を優先するため |
